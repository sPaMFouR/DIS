Introduction to 'pandas'¶

by Kaustubh Vaghmare¶

Dealing with Data¶

There are several aspects of dealing with data.

  • You need to know how to take your data from a file or a web resource and represent it in your program.
  • Your data need not be in a form required to perform various analysis - you need to tidy it up.
  • Your data can be a mixture of high quality data, low quality and missing data.
  • Your data is in one form but the intended analysis algorithm needs it in another form.
  • Sometimes units are not correct or uniform.
  • Data contains duplicate entries owing to the procedure of how it was recorded.

How do we deal with these issues?

Definitely not in Pure Python¶

You can in principle use Python libraries to take data in a string form from almost any resource around the world. Then if-else and while will indeed help you to get your data organized in a structure amenable to analysis but there are several problems

  • Performance in general of such routines will be poor.
  • You will end up struggling more with code and have left no energy for actual analysis.

Numpy is better¶

It provides high performance arrays to organize data and further gives several methods to query these arrays for useful information. But even here, you will spend more time maintaining relationships between various arrays than the actual data.

Enter Pandas¶

Pandas builds on top of Numpy and provides even more sophisticated data structures so that you spend very little time with technical aspects and more on the data.

So, what is pandas?¶

It is an open source Python library developed by Wes McKinney.

  • It offers various data structures to represent structured data.
  • Provides several functions commonly needed for handling such data.
  • Gives several methods to retrieve data from various sources.
  • Provides high level interface to matplotlib to take care of several visualization aspects for you.
  • And much more...

The 'Series'¶

Let us begin with the building block in Pandas, known as Series. Let us start with a Numpy array first.

In [97]:
a = np.array([1,4,64,23,11])
print(a[0]) # access by position
b = np.array([54,23,132,23,12]) # another array
print(a+b) # position matched arithmetic

1
[ 55  27 196  46  23]

Now, consider a sort of a Numpy array where one can obviously access by position but also through 'labels'. Further, when performing operations between two such data structures, it is not the position but the label which matters. That's a Series.

In [98]:
# Let's make a Series from a list.
s = Series([1,2,3,4,5])
print(s)

0    1
1    2
2    3
3    4
4    5
dtype: int64

What is that column on the left? These are the labels assigned to each of our value. Since we did not specify any label, pandas chose integer sequence starting from 0 to N-1 where N is number of integers.

What is that column on the right? Those are our actual values.

Notice that pandas can see we have supplied all integers and chooses the int64 Numpy data type for our representation.

The way we have constructured our first Series, it almost seems like a Numpy array. Time to change that. Let's specify our own labels now.

Alignments¶

In [117]:
s1 = Series([1,2,3,4,5], index=['a','b','c','d','e'])
s2 = Series([1,2,3,4,5], index=['b','c','d','e','f'])
series_sum = s1+s2
print(series_sum)

a    NaN
b    3.0
c    5.0
d    7.0
e    9.0
f    NaN
dtype: float64

Unlike Numpy arrays, series don'y simply add by position. They are aligned by their labels and then added up. If any label is missing in one and present in another, it becomes NaN.

Behavior of NaNs in Pandas¶

In [118]:
series_sum_array = np.array(series_sum)
print(series_sum_array.sum())

nan
In [119]:
print(series_sum.sum())
print(series_sum.mean())

24.0
6.0

So, Numpy takes operation on elements very seriously and strictly - NaN + anything is a NaN. Pandas is lenient. It ignores NaNs and operates on the rest of the array. That is what a data analyst does need.

DataFrames¶

While a Series object is a structure for 1-d data, DataFrames are structures meant for 2-d data. One way to think about them is a dictionary of Series with keys representing 'column names' and a common index for all the Series objects. Another way to think about them is a 2-d matrix with an index marking the rows and column names marking the columns.

Even from the point of view of making a DataFrame object, both these ways of visualizing a DataFrame hold good. Let's see.

Loading Data¶

So far we have been constructing dataframes from random number generators. This is obviously for illustration purposes. But in reality, data comes from various resources such as

  • Files on your computer
  • The web
  • An API to a data service
  • Database servers etc.

Let us very quickly review the first two. The remaining two are beyond the scope of this presentation.

A CSV File¶

df = pd.read_csv('path/to/file.csv')

Do you want a specific column to be used as an index?

df = pd.read_csv('path/to/file.csv', index_col=0) # uses 1st column as index

Pandas will automatically determine data types for each column. Want to specify your own?

df = pd.read_csv('path/to/file.csv', dtype={'colName': np.float64})

Wish to give your own names to the columns?

df = pd.read_csv('path/to/file.csv', header=0, names=['one', 'two',...])

Load only specific columns.

df = pf.read_csv('path/to/file.csv', use_cols=['col1', 'col2'])

A general ASCII file.¶

df = pd.read_table('path/to/file', sep='|')

Have a footer of 4 lines which you need to ignore?

df = pd.read_table('path/to/file', sep='|', skip_footer=4)

Load only 5 rows?

df = pd.read_table('path/to/file', sep='|', nrows=3)

And more ... check documentation.

Saving files from dataframes¶

Whether it's a CSV file or a plain ASCII file, the procedure for saving the file is same.

df.to_csv('file)

For generic ASCII

df.to_csv('file', sep='|')

By default, to_csv() will discard index information. To store it as a file,

df.to_csv('file', index_label='someColName')

From and to an Excel sheet.¶

df = pd.read_excel('file.xls', sheetname='Sheet1')

df.to_excel('some.xls')

From and to a JSON file dump.¶

df = pd.read_json('file.json')

df.to_json('file.json')

From and to HTML pages.¶

df = pd.read_html('filename OR URL')

df.to_html('some.html')

Pandas has some in-built tools to directly access some commonly used web data resources. These include Yahoo! Finance for example. Refer to documentation for more details.

Pandas can directly interact with a database through a connection established using the standard Python API. Again, refer to documentation for more details.

Visualization¶

Pandas does not have visualization capabilities of its own. However, it does interact with 'matplotlib' and produces very useful visualizations for you. It uses the information about the data captured in a DataFrame and does a lot of work for you - such as choosing the right axis labels, the labels for the legends etc.

Especially if you are using a Notebook, it is better to set some options before we get started.
